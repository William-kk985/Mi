# 开发进度

## 环境

- 仿真容器：`cyberdog_sim:ready`，启动脚本 `./start_sim.sh`
- 宿主机 `src/` 挂载进容器，代码修改实时同步
- 编译（需先启动容器）：
  ```bash
  # 1. 启动容器（如果没跑）
  docker start cyberdog

  # 2. 进入容器编译
  docker exec -it cyberdog bash -c "cd /home/cyberdog_sim && source /opt/ros/galactic/setup.bash && colcon build --merge-install --packages-select cyberdog_race"

  # 3. 更新镜像（编译成功后执行）
  docker commit cyberdog cyberdog_sim:ready
  ```

## 已完成内容

### 基础框架
- 状态机主控节点 `race_controller`，100Hz 控制循环
- 6 个赛段类（Stage1~6），统一 `StageBase` 接口
- 调试宏体系 `debug_config.hpp`
  - `DEBUG_VISION` 视觉 imshow 可视化
  - `DEBUG_MOTION` 运动指令日志
  - `DEBUG_SENSOR` 传感器数据日志
  - `DEBUG_STAGE` 状态机切换日志
  - `DEBUG_SINGLE_STAGE N` 只跑第 N 赛段
  - `DEBUG_START_STAGE N` 从第 N 赛段开始

### Bug 修复
- IMU 订阅 QoS 改为 `BEST_EFFORT`，匹配仿真器发布者
- IMU 四元数正确转换欧拉角（`atan2` 方法，修复原来直接取 `orientation.z` 导致转圈的问题）
- `RCLCPP_INFO_THROTTLE` 改用 `static rclcpp::Clock`，修复每帧创建临时 Clock 导致的崩溃

### MotionCtrl 新增接口
- `set_pitch(float)` 控制身体俯仰角，`rightStickAnalog[1]`，负值低头，范围约 [-0.9, 0.9]

### 第一赛段
- 黄线巡线：HSV 分割 + 逐行扫描 + PD 控制
- 单边丢失时用历史车道宽度估算中心线
- 路口触发：里程计到达 `(3.0, 0.17)` 附近 0.15m 范围内触发（去掉了 Lidar 条件）
- 转向：P 控制对齐到 yaw = `-π/2`，误差 < 0.05rad 停止
- 低头/抬头：`init()` 时 `set_pitch(-0.4)` 低头看黄线，转向完成后 `set_pitch(0)` 恢复
