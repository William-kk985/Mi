# 2026小米杯 CyberDog 赛题方案设计

## 整体架构

- 状态机驱动，每个赛段为一个状态，赛段结束条件触发后自动切换
- 视觉主传感器：RGB_camera（巡线/颜色识别）+ D435_color（目标识别）+ D435_depth（测距/避障）
- 运动控制：LCM gamepad_lcmt 指令，yaw 用 rightStickAnalog[0]，前进用 leftStickAnalog[1]
- 参考智能车传统视觉方案：HSV 颜色分割 + 逐行扫描 + 贝塞尔中心线 + PD 控制

---

## 第一赛段：石径探路

### 场景特征
- 石板路：宽 30cm，高 5cm，间隔 20cm
- 赛道边沿：黄色（RGB 255,255,0），宽 15cm（弯道 10cm）
- 需找到第一个弯道，完成拐弯后进入第二赛段
- 本赛段不允许跳过

### 难点
1. 石板高 5cm + 间隔 20cm，容易绊倒
2. 弯道处单侧边界消失，需切换单边跟踪
3. 赛段结束判断需准确，避免提前切换

### 方案
- **视觉**：HSV 提取黄色边界（H: 20-35, S>100, V>100），逐行扫描左右边界，计算中心线
- **控制**：中心线偏差 → PD 控制 → rightStickAnalog[0] 修正 yaw
- **石板应对**：调高 cyberdog2-ctrl-user-parameters.yaml 中步高参数，增大抬腿高度
- **弯道检测**：右边界斜率方差突变（>80）且左边界消失 → 判断为右弯，切换单边跟踪
- **赛段结束**：IMU 偏航角累计转过约 90° + 视觉重新看到双边界 → 切换第二赛段

## 第二赛段：荒野寻珠

### 场景特征
- 4×4 球阵（C1-C4，R1-R4），列间距 100cm，行间距 64cm
- 入口处 R4C4、R4C3、R3C4 固定蓝色，其余每行每列各 1 个橙色球（共 4 个）
- 球底部距地面 20cm，球直径 20cm，球心高度约 30cm
- 出口在左上角

### 难点
1. 球位置赛前随机调整，需实时识别
2. 里程计在 3m×2.5m 范围内会有漂移
3. 撞完所有橙色球后需准确走到左上角出口

### 基础方案：S 型扫描路径（已实现）

**坐标系说明（实测确认）：**
- yaw π/2 ≈ 1.57 = y轴正方向（进入球阵方向，向上）
- yaw 0 = x轴正方向（向右）
- yaw π/-π = x轴负方向（向左）
- set_velocity yaw 正值 = 右转（yaw减小）
- set_velocity yaw 负值 = 左转（yaw增大）

**路径规划（通道导航，不经过球位）：**
- 入口进来先向上走到 y=1.1，再按 S 型扫描
- 右侧通道 x=2.7，左侧通道 x=0
- y 路径点依次：1.1 → 1.7 → 2.6 → 3.5 → 4.2（出口）
- 路径走球列之间通道，避免惯性撞上蓝球

**球阵坐标参考（仿真实测）：**
- 列间距 1.2m，行间距 0.8m
- C4: x=3.2，C3: x=2.0，C2: x=0.8，C1: x=-0.4
- R1: y=1.3，R2: y=2.1，R3: y=2.9，R4: y=3.7

**扫描逻辑：**
- 到达每个路径点后原地左右转 ±40°
- 发现橙色球且距离 < 0.5m 才冲击（避免误判远处球）
- 冲击后退回路径点继续

**出口：**
- 所有点扫完后走到 (-0.15, 4.2)，赛段结束

## 第三赛段：曲道冲锋

### 场景特征
- 曲线-直线-曲线 S 形赛道，净宽 100cm
- 全程不得踩线或越线
- 抵达右上角出口，后腿足底离开出口虚线后赛段结束
- 无随机元素，赛道结构固定

### 方案
- 直接复用第一赛段黄线巡线逻辑：HSV 分割 + 逐行扫描 + 贝塞尔中心线 + PD 控制
- 弯道检测：斜率方差 > 80 时判断进入弯道，自动降速
- 弯道时适当加大 yaw 响应增益，转向更灵敏
- 出口检测：单侧黄线消失 + IMU 偏航角变化 → 赛段结束，切换第四赛段

### 注意事项
1. 机器狗转弯时身体有侧倾，实际占用宽度比静止时略大，弯道速度不宜过快
2. 弯道内外侧黄线曲率不同，中心线计算容易偏向内侧，需验证中心线是否居中
3. S 形有两个方向相反的弯，第二个弯进入时 yaw 方向相反，控制器符号需正确处理
4. 赛道净宽 100cm，机器狗体宽约 30cm，容错空间约 35cm/侧，正常巡线不会踩线
5. 直线段可以适当提速，弯道前提前减速，节省整体时间

## 第四赛段：深隧寻珍

### 场景特征
- 1 条横向通道 + 3 条竖向通道（C1、C2、C3）
- 3 个目标物（位置赛前随机，在竖向通道 1m 范围内）：
  - 大桶可乐（2L，透明瓶）
  - 橙色球（底部距地 60cm，球心约 70cm）
  - 足球（黑白配色，直径 20cm）
- 3 个障碍物：限高杆 x2（红色，底部 40cm）、方块障碍物 x1（20cm³，间隔 20cm）
- 球框：50cm×30cm，深度 50cm
- 赛段结束：前腿足底碰到独木桥起始端

### 方案

**通道导航：固定顺序 C1→C2→C3**
- 沿横向通道行走，里程计计数，到达各竖向通道入口时转向进入
- 每个竖向通道搜索完毕后退回横向通道，继续前进

**目标识别：**
- 可乐瓶：YOLO 识别（透明圆柱，HSV 无法区分）
- 橙色球：HSV 橙色分割（H 10-20, S>150, V>150），高度 60cm 需仰视
- 足球：YOLO 识别（黑白纹理）
- 识别成功后语音播报对应文字

**障碍处理：**
- 限高杆（红色，底部 40cm）：机器狗正常身高 25cm，天然能通过，无需降低身体
  - 识别到红色横杆（HSV 红色）→ 语音播报"识别到限高杆" → 正常通过
- 方块障碍物：Lidar 检测 → 绕行，可穿过虚线借道

**目标交互：**
- 可乐瓶：对准后加速冲击撞倒
- 橙色球：对准后冲击使其明显晃动（不允许后空翻）
- 足球：D435 深度测距，对准球门方向，慢速推进，连续小步推入球门

**参数确认：**
- 正常身体高度：`des_roll_pitch_height: [0.0, 0, 0.25]`（25cm）
- 限高杆底部 40cm > 机器狗身高 25cm，无需调整高度参数

### 注意事项
1. 目标物和障碍物位置随机，不能依赖固定坐标，必须靠视觉实时识别
2. 橙色球高度 60cm，相机需要仰视，注意相机视角范围
3. 足球推入球门时保持低速，避免推偏，可连续多次小步推
4. 限高杆全程不能碰，识别到后保持直行通过即可
5. 完成三个目标后需找到独木桥起始端，注意独木桥在赛段右下角

## 第五赛段：孤梁稳渡

### 场景特征
- 连续独木桥：直行段 → 左转 → 方形区域，全程在桥上行走
- 四个足底都越过终点虚线后才可以跳下
- 跳下时不允许身体撞击独木桥
- 赛段结束：成功跳下

### 方案
- 上桥：低速前进，切换慢速稳定步态
- 桥上居中（双重保障）：
  - 主：复用第三赛段黄线巡线逻辑，HSV 提取桥面左右黄色边缘，逐行扫描计算中心线，PD 控制修正 y 方向速度
  - 辅：D435 深度点云检测桥面存在性，走偏时兜底报警
  - 注意：桥面视角与第三赛段不同（机器狗在桥上而非桥间），扫描参数需重新标定
- 直行段：低速居中前进
- 转弯检测：黄线分布变化（单侧黄线消失或方向偏转）→ 触发左转，转完继续居中行走
- 方形区域：同样黄线巡线居中行走
- 虚线检测：里程计计距为主，相机识别虚线为辅，确认四足越线后执行跳下
- 跳下：发跳跃指令向前跳离桥面，落地后切换第六赛段

### 注意事项
1. 转弯时在独木桥上，横向容错最小，转弯速度要慢
2. 跳下方向要向前，避免落地时后腿撞到桥面
3. 四足必须全部越过虚线才能跳，用足底接触传感器辅助确认

## 第六赛段：撷金建功

### 场景特征
- 从独木桥跳下后在地面区域
- 将足球从出口位置踢出，走到终点位置四足在圈内趴下
- 设备摔倒或碰撞独木桥不扣分，小球从独木桥上飞出后不允许放回

### 方案

**出口定位：Lidar**
- 第六赛段区域四周是独木桥桥体，出口是一个缺口
- Lidar 180° 扫描：有障碍物方向 = 桥体，空旷方向 = 出口
- 取 Lidar 扫描中最大空旷角度方向作为出口方向

**找球：相机**
- 落地稳定后，用 YOLO 或 HSV 黑白分割定位足球
- D435 深度测距确认球的距离

**带球到出口：Lidar + 相机双传感器配合**
- Lidar 持续确认出口方向
- 相机持续跟踪球的位置
- 每推一小步检查球是否还在出口方向连线上
  - 球偏了 → 绕到球侧面重新对准再推
  - 方向对 → 继续低速推进
- 推到出口附近（Lidar 前方空旷距离突然增大）→ 加速踢出

**终点：**
- 球踢出后，转向找终点圈
- 走入终点圈，确认四足在圈内，发趴下指令结束比赛

### 注意事项
1. 落地后先稳定姿态再开始找球，避免落地冲击影响视觉
2. 推球全程低速，球滚偏时及时绕到侧面修正，不要强推
3. Lidar 找出口是关键，出口方向确定后推球方向就确定了
4. 终点圈四条腿足底必须全部在圈内才算完成
5. 摔倒不扣分，推球稳定优先于速度
