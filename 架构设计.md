# cyberdog_race 代码架构设计

## 包结构

位置：`cyberdog_sim/src/cyberdog_simulator/cyberdog_race/`（与 cyberdog_example 同级）

```
cyberdog_race/
├── CMakeLists.txt
├── package.xml
├── include/cyberdog_race/
│   ├── motion_ctrl.hpp        # LCM指令封装
│   ├── vision/
│   │   ├── lane_detector.hpp  # 黄线巡线（第1/3/5赛段复用）
│   │   ├── ball_detector.hpp  # HSV球识别（橙色/蓝色）
│   │   └── yolo_detector.hpp  # YOLO识别（可乐瓶/足球）
│   └── stages/
│       ├── stage_base.hpp     # 基类：init()/run()/is_done()
│       ├── stage1.hpp ~ stage6.hpp
└── src/
    ├── main.cpp               # 主节点 + 状态机
    ├── motion_ctrl.cpp
    ├── vision/
    │   ├── lane_detector.cpp
    │   ├── ball_detector.cpp
    │   └── yolo_detector.cpp
    └── stages/
        └── stage1.cpp ~ stage6.cpp
```

## 线程模型

```
主线程（100Hz）                视觉线程（30Hz）
  状态机切换              ←→   相机图像处理
  运动控制输出                  结果写入共享变量
  读取视觉结果                  lane_result / ball_result / detect_result
```

- 主线程负责状态机和 LCM 运动指令，高频稳定
- 视觉线程独立处理图像，不阻塞主线程
- 共享数据用 mutex 保护

## 核心接口

### stage_base
```cpp
class StageBase {
public:
    virtual void init() = 0;
    virtual void run() = 0;
    virtual bool is_done() = 0;
};
```

### motion_ctrl
```cpp
void set_velocity(float x, float y, float yaw);
void stand();
void lie_down();
void recovery_stand();
```

## 扩展原则
- 新增赛段：继承 StageBase，只改对应 stage 文件
- 替换视觉算法：只改 vision/ 下对应模块，接口不变
- 主线程和视觉线程解耦，互不影响
